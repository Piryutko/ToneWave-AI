# ToneWave AI 🌊

> *Ваш личный дипломат: превращает эмоции в конструктив*

**ToneWave AI** — это веб-приложение, которое работает как ваш личный ИИ-переводчик. Оно использует локальные нейросети (через [Ollama](https://ollama.com/)) для преобразования резких, жестких или нецензурных высказываний в вежливый и профессиональный текст, полностью сохраняя при этом изначальный смысл. 

## 🚀 Возможности

Приложение предлагает современный, реактивный веб-интерфейс с поддержкой 4 режимов трансформации:
- 👔 **PROFESSIONAL**: Деловой стиль и конструктив. Идеально для бизнес-переписки.
- 💖 **KIND**: Максимальная эмпатия и поддержка. Превращает претензию в добрую и мягкую просьбу.
- ⚙️ **TECHNICAL**: Строгая и сухая аналитика. Удаляет эмоции, оставляя чистые факты.
- ⚡ **LACONIC**: Сжатие до одной мощной и емкой фразы (5-7 слов) с сохранением главного посыла.

*Ключевая фича:* Трансформированный текст "печатается" на экране в реальном времени, сразу по мере генерации ответа локальной нейросетью.

## 🏗 Архитектура

Проект разделен на две основные части:

1. **`ToneWave.Api` (Backend)** 
   - Написан на **C# 12 / .NET 8.0 Minimal API**.
   - Отвечает за коммуникацию с локальной установкой Ollama.
   - Использует технологию **Server-Sent Events (SSE)** для потоковой передачи сгенерированных ИИ токенов клиенту.

2. **`tonewave-web` (Frontend)**
   - Написан на **SvelteKit** + **TypeScript**.
   - Стилизован с помощью нового **Tailwind CSS v4** (использует Glassmorphism).
   - Читает SSE-поток от бекенда и реактивно отображает результат.

*(Также в репозитории осталась папка `ToneWaveCLI` — это ранняя консольная версия приложения).*

## 📦 Установка и запуск

### 1. Подготовка Ollama
Убедитесь, что у вас работает [Ollama](https://ollama.com/) и локально скачана модель `llama3.2`.
Для загрузки модели выполните в терминале:
```bash
ollama run llama3.2
```

### 2. Запуск Backend (ToneWave.Api)
Перейдите в папку бекенда и запустите приложение на .NET:
```bash
cd ToneWave.Api
dotnet run
```
Бекенд запустится на порту `http://localhost:5000`.

### 3. Запуск Frontend (tonewave-web)
В новом окне терминала перейдите в папку фронтенда, установите зависимости и запустите dev-сервер:
```bash
cd tonewave-web
npm install
npm run dev
```
Фронтенд будет доступен по адресу `http://localhost:5173`. 

Откройте этот адрес в браузере, и ToneWave AI готов к работе!

## 🗺️ Roadmap & Future Improvements

Этот план развития описывает возможные пути улучшения и расширения проекта ToneWave AI.

### 📌 Phase 1: Улучшение пользовательского опыта (UX/UI)
- [ ] **История запросов:** Сохранение истории транформаций в Local Storage (чтобы пользователь мог посмотреть, что он переводил вчера).
- [ ] **Кнопка "Очистить":** Быстрая очистка полей ввода и вывода.
- [ ] **Тост-уведомления (Toasts):** Красивые всплывающие сообщения об успешном копировании текста или возникших ошибках, вместо системных алертов.
- [ ] **Поддержка Markdown:** Если Ollama возвращает списки или жирный текст, отрендерить их на Svelte красиво (например, через `marked` или `mdsvex`).
- [ ] **Темная/Светлая тема:** Добавить переключатель тем (хотя текущий "стеклянный" темный дизайн выглядит очень круто).

### 🚀 Phase 2: Расширение функционала AI (ToneWave.Api)
- [ ] **Кастомные промпты:** Позволить пользователю вводить свои инструкции (например, "Переведи на английский вежливо" или "Сделай текст в стиле Шекспира").
- [ ] **Выбор моделей:** Если у вас скачано несколько моделей (llama3.2, mistral, qwen), добавить выпадающий список на фронтенд для переключения между ними "на лету".
- [ ] **Регулировка "Температуры":** Ползунок на фронтенде (Креативность vs Точность), который передается в API.
- [ ] **Анализ тональности:** Перед трансформацией выводить, насколько "токсичным" был оригинальный текст (по шкале от 1 до 10).

### 🛠 Phase 3: Архитектура, Стабильность и Деплой
- [ ] **Rate Limiting:** Добавить ограничение количества запросов к бекенду (чтобы кто-то не "положил" ваш домашний комп, если вы расшарите ссылку).
- [ ] **Централизованное логирование:** Использовать Serilog для записи логов в файл, чтобы было проще отлаживать ошибки.
- [ ] **Docker:** Упаковать бекенд `ToneWave.Api` в Docker-контейнер для более простого запуска на любых серверах (VPS).
- [ ] **Автоматизация Cloudflare:** Настроить Cloudflare Tunnel как системную службу (service), чтобы не нужно было каждый раз вручную держать терминал открытым.

### 📱 Phase 4: Новые платформы
- [ ] **Telegram Бот:** Сделать бота, которому можно переслать гневное сообщение, а он в ответ пришлет вежливый вариант (используя всё тот же `ToneWave.Api`).
- [ ] **Браузерное расширение (Chrome/Edge):** Выделяешь текст на любом сайте, нажимаешь правую кнопку мыши -> "Улучшить тон".
